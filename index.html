Relay Testing Procedures

Pre-Commissioning Checks

1. Verify CT/VT polarity using phase rotation test
2. Check binary input status (CB, isolator positions)
3. Test trip circuit continuity
4. Verify communication with SCADA/substation controller
5. Review event logs for anomalies

Primary Injection Testing

‚ö† Safety First: Always follow lockout/tagout procedures. Verify isolation before testing.

‚Ä¢ Secondary injection test set (Omicron CMC, Megger SMRT)
‚Ä¢ Multimeter and clamp meter
‚Ä¢ Laptop with DIGSI software

Overcurrent Test Procedure

1. Set relay to test mode (disable trip outputs)
2. Inject current at 1.1 x I> pickup setting
3. Verify relay detects and times correctly
4. Record actual vs. expected operating time
5. Repeat for I>> and I>>> elements

Secondary Injection Testing

Using test sets like Omicron CMC or Megger SMRT for precise relay testing.

Time-Overcurrent Curve Testing

Test Points (multiples of pickup):
1.5x, 2x, 3x, 5x, 10x, 20x

Tolerance: ¬±5% of setting

Directional Element Test

  • Apply voltage and current with varying phase angles

IEC 61850 Configuration

IEC 61850 is the international standard for substation automation systems enabling interoperability between devices from different manufacturers.

Key Concepts

  • Logical Nodes (LN): Standardized functions (XCBR for breaker, PDIF for differential)
  • Data Objects: Information within logical nodes
  • GOOSE: Generic Object Oriented Substation Event (fast peer-to-peer)
  • MMS: Manufacturing Message Specification (client-server communication)
  • SCL: Substation Configuration Language (XML configuration files)

SCL File Types

File Type Purpose
ICD IED Capability Description - device template
CID Configured IED Description - configured device
SCD Substation Configuration Description - complete system
IID Instantiated IED Description - device with data values

Fault Type Identification

Single Line-to-Ground (L-G):
‚Ä¢ One phase current increases significantly
‚Ä¢ Ground current appears
‚Ä¢ Affected phase voltage drops to near zero
‚Ä¢ Other phases show minimal voltage rise
Line-to-Line (L-L):
‚Ä¢ Two phase currents increase equally (opposite polarity)
‚Ä¢ No ground current
‚Ä¢ Both affected phase voltages drop
‚Ä¢ Third phase unaffected
Three-Phase (3œÜ):
‚Ä¢ All three phase currents increase equally
‚Ä¢ Balanced fault (symmetrical)
‚Ä¢ All phase voltages collapse
‚Ä¢ Highest fault current magnitude

Advanced Analysis Features

Harmonic Analysis

Use FFT (Fast Fourier Transform) to identify harmonic content during faults.

  • 2nd Harmonic: Indicates transformer inrush (not a fault)
  • 3rd Harmonic: Common in ground faults
  • 5th/7th Harmonic: Power electronics, VFDs

Sequence Component Analysis

Positive Sequence (I1): Normal balanced load
Negative Sequence (I2): Unbalanced faults (L-L)
Zero Sequence (I0): Ground faults (L-G)
Fault Type Detection:
‚Ä¢ L-G fault: I0 present, I2 moderate
‚Ä¢ L-L fault: I2 high, I0 absent
‚Ä¢ 3œÜ fault: Only I1 present

CFC Logic Programming Examples

Practical logic schemes commonly implemented in SIPROTEC relays using CFC (Continuous Function Chart).

1. Bus-Tie Interlocking Scheme

Prevents closing bus-tie breaker when both sources are energized with out-of-sync conditions.

Logic Flow:

‚îÇ Sync_Check_OK ‚îÇ
‚îî‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îò
Output ‚Üí Bus_Tie_Close_Enable

CFC Implementation

1. Create binary inputs for Bus1_OK, Bus2_OK
2. Add SYNCHK function block (synchronism check)
3. Configure voltage limits (¬±10%), frequency (¬±0.2 Hz), angle (¬±20¬∞)
4. Use 3-input AND gate to combine conditions
5. Output to close permit relay
Settings:
‚Ä¢ Voltage difference: ¬±10% of nominal
‚Ä¢ Frequency difference: ¬±0.2 Hz
‚Ä¢ Phase angle: ¬±20¬∞
‚Ä¢ Dead bus bypass: Enable (allow close if one bus dead)

2. Automatic Transfer Scheme (ATS)

Automatically transfers load from failed source to alternate source.

Sequence:
Source1_Fail ‚Üí Delay 0.5s ‚Üí Trip CB1 ‚Üí
Delay 1.0s ‚Üí Close CB2 (if Source2_OK)

CFC Components

1. Source monitoring: Undervoltage elements (27)
2. Timer 1: Dropout delay (0.5s) before tripping CB1
3. Timer 2: Dead-time delay (1.0s) before closing CB2
4. SR Flip-Flop: Maintain transfer state
5. AND gates: Combine permissive conditions
‚ö† Critical Checks:
‚Ä¢ Verify source 2 is available before transfer
‚Ä¢ Ensure CB1 is fully open before closing CB2
‚Ä¢ Prevent rapid re-transfer (add time delay)
‚Ä¢ Include manual override capability

3. Load Shedding Scheme

Automatically shed non-critical loads during underfrequency or overload conditions.

Multi-Stage Load Shedding

Stage Frequency Delay Load %
1 59.5 Hz 0.3s 10%
2 59.0 Hz 0.3s 15%
3 58.5 Hz 0.2s 20%
4 58.0 Hz 0.1s 25%

CFC Logic

1. Use COMPARATOR blocks to detect frequency levels
2. Add independent TIMER for each stage
3. Use SR flip-flop to latch trip command
4. Include frequency restoration logic (reset at 59.8 Hz)
5. Add manual load restoration capability

4. Breaker Failure Protection

Backup protection when circuit breaker fails to clear fault.

Logic:
Trip_Command AND Current_Still_Present
‚Üí Start Timer (150ms typical)
‚Üí Timer Expires ‚Üí Trip Adjacent Breakers

Implementation Steps

1. Monitor trip command from protection functions
2. Check if fault current persists after trip
3. Start breaker failure timer (100-200ms)
4. If current still present when timer expires ‚Üí trip backup
5. Send trip to adjacent breakers (zone 2 backup)
Current Supervision:
Set pickup at 25-50% of minimum fault current to ensure detection while avoiding misoperation during load conditions.

5. Anti-Pumping Logic

Prevents repeated close attempts when fault is still present.

IF (Close_Command AND Trip_Condition_Active)
THEN Block_Close
UNTIL Trip_Condition_Cleared

CFC Design

1. Monitor all trip signals (protection + manual)
2. Use OR gate to combine all trip sources
3. Set flip-flop on any trip
4. Block close command while flip-flop set
5. Reset only when trip condition clears

6. Voltage-Controlled Overcurrent

Increase sensitivity of overcurrent protection during low voltage conditions.

Logic Concept:
Normal: I> = 1.2 x Rated Current
Low Voltage (<85%): I> = 0.8 x Rated Current

CFC Implementation

1. Use COMPARATOR to detect voltage < 85%
2. Create two overcurrent elements with different settings
3. Use MUX (multiplexer) to select active element based on voltage
4. Route selected output to trip logic

CFC Best Practices

  • Use clear, descriptive names for all signals and blocks
  • Group related logic in separate CFC charts
  • Add comments explaining complex logic sequences
  • Test in simulation mode before downloading
  • Document all custom logic in commissioning reports
  • Use consistent color coding for different signal types
  • Minimize crossing wires for readability
  • Always include manual override for automatic functions

    Relay Operation Issues

    Unwanted Tripping

    Diagnostic Steps:
    1. Review fault records and event logs
    2. Check if trip was from protection function or external signal
    3. Verify CT/VT inputs are correct (no open circuits)
    4. Analyze pre-fault and fault conditions
    5. Check for incorrect settings or logic programming

    Failure to Trip

    Diagnostic Steps:
    1. Verify protection function is enabled
    2. Check CT/VT input signals are present
    3. Test trip circuit continuity
    4. Review pickup and time delay settings
    5. Check if relay detected fault (event log)
    6. Verify trip logic in CFC (if applicable)
